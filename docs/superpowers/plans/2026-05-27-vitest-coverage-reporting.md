# Vitest Coverage Reporting Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add V8-based Vitest coverage reporting and make coverage thresholds part of the local check gate.

**Architecture:** Keep normal unit tests unchanged and mocked by default. Add coverage settings to the existing Vitest config so `pnpm run coverage` reports only `src/**/*.js`, then wire `pnpm run check` to use coverage instead of a duplicate plain test run.

**Tech Stack:** Node.js ESM, pnpm 11, Vitest 4, `@vitest/coverage-v8`, Biome.

### Task 2: Vitest Coverage Configuration

**Files:**
- Modify: `vitest.config.js`

- [ ] **Step 1: Configure coverage**

Update `vitest.config.js` to:

```js
import { defineConfig } from "vitest/config"

export default defineConfig({
  test: {
    coverage: {
      provider: "v8",
      include: ["src/**/*.js"],
      reporter: ["text", "html", "lcov"],
      thresholds: {
        lines: 80,
        statements: 80,
        functions: 80,
        branches: 70,
      },
    },
    environment: "node",
    include: ["tests/**/*.test.js"],
    restoreMocks: true,
  },
})
```

- [ ] **Step 2: Run coverage**

Run: `pnpm run coverage`

Expected: Coverage runs with `text`, `html`, and `lcov` reporters. If thresholds fail, inspect the uncovered files and add focused tests for meaningful source behavior instead of lowering thresholds.
