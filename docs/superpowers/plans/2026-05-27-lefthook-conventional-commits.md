# Lefthook Conventional Commits Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add modern local Git hooks that run Biome lint before commits and enforce Conventional Commit messages.

**Architecture:** Use Lefthook as the hook runner and commitlint as the Conventional Commit validator. Keep hook behavior in `lefthook.yml`, commit-message rules in `commitlint.config.js`, and document the workflow in contributor/development docs.

**Tech Stack:** Node.js ESM, pnpm 11, Lefthook, commitlint, Biome.

---

## File Structure

- Modify `package.json`: add `prepare` and `commitlint` scripts plus dev dependencies.
- Modify `pnpm-lock.yaml`: generated by `pnpm add -D lefthook @commitlint/cli @commitlint/config-conventional`.
- Modify `pnpm-workspace.yaml`: allow Lefthook's pnpm 11 build script so hooks install cleanly.
- Create `lefthook.yml`: define `pre-commit` and `commit-msg` hooks.
- Create `commitlint.config.js`: extend `@commitlint/config-conventional`.
- Modify `DEVELOPMENT.md`: document hook installation and local validation commands.
- Modify `CONTRIBUTING.md`: document Conventional Commit and branch naming expectations.

### Task 2: Hook Configuration

**Files:**
- Create: `lefthook.yml`
- Create: `commitlint.config.js`

- [ ] **Step 1: Configure commitlint**

Create `commitlint.config.js`:

```js
export default {
  extends: ["@commitlint/config-conventional"],
}
```

- [ ] **Step 2: Configure Lefthook hooks**

Create `lefthook.yml`:

```yaml
pre-commit:
  commands:
    lint:
      run: pnpm run lint

commit-msg:
  commands:
    commitlint:
      run: pnpm run commitlint --edit {1}
```

Expected: Lefthook runs Biome before commits and commitlint on the commit message file.

### Task 3: Documentation

**Files:**
- Modify: `DEVELOPMENT.md`
- Modify: `CONTRIBUTING.md`

- [ ] **Step 1: Document local hooks**

Add a development section that says `pnpm install` runs `lefthook install`, `pnpm run lint` is the pre-commit gate, and commit messages are checked with commitlint.

- [ ] **Step 2: Document naming conventions**

Add contributor guidance for Conventional Commits and branch prefixes:

```text
feat: add a user-visible capability
fix: correct a bug
docs: update documentation only
test: add or update tests
chore: update maintenance tooling
ci: update automation
```

Branch examples:

```text
feat/telegram-voice-cache
fix/album-cleanup
chore/update-tooling
```
